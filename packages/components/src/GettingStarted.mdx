# Getting Started with Lean IDS

This guide will help you install and set up Lean IDS in your React application.

---

## 📋 Prerequisites

Before you begin, ensure you have:

- **Node.js** >= 18.0.0
- **npm** >= 9.0.0 or **yarn** >= 1.22.0
- **React** >= 18.0.0
- **React DOM** >= 18.0.0

---

## 📦 Installation

### Step 1: Install the Package

Install Lean IDS components and tokens in your project:

```bash
npm install @lean-ids/components @lean-ids/tokens
```

Or with yarn:

```bash
yarn add @lean-ids/components @lean-ids/tokens
```

### Step 2: Install Peer Dependencies

Lean IDS requires React and styled-components:

```bash
npm install react react-dom styled-components
```

Or with yarn:

```bash
yarn add react react-dom styled-components
```

---

## 🎨 Basic Setup

### Step 1: Wrap Your App with ThemeProvider

The `ThemeProvider` component provides theme context to all Lean IDS components.

```tsx
import React from 'react';
import { ThemeProvider } from '@lean-ids/components';

function App() {
  return (
    <ThemeProvider theme="carelon">
      {/* Your app content */}
    </ThemeProvider>
  );
}

export default App;
```

### Step 2: Import and Use Components

Import components from `@lean-ids/components`:

```tsx
import React, { useState } from 'react';
import { 
  ThemeProvider, 
  Button, 
  InputField, 
  Checkbox 
} from '@lean-ids/components';

function MyForm() {
  const [email, setEmail] = useState('');
  const [agreed, setAgreed] = useState(false);

  return (
    <ThemeProvider theme="carelon">
      <form>
        <InputField
          label="Email Address"
          type="email"
          value={email}
          onChange={(e) => setEmail(e.target.value)}
          placeholder="Enter your email"
        />
        
        <Checkbox
          label="I agree to the terms and conditions"
          checked={agreed}
          onChange={(e) => setAgreed(e.target.checked)}
        />
        
        <Button 
          variant="primary" 
          size="medium"
          disabled={!agreed}
        >
          Submit
        </Button>
      </form>
    </ThemeProvider>
  );
}

export default MyForm;
```

## 🎭 Theme Configuration

### Available Themes

Lean IDS supports multiple brand themes:

- **carelon** - Carelon brand theme (default)
- **elevance** - Elevance Health brand theme

### Switching Themes

```tsx
import { ThemeProvider } from '@lean-ids/components';

// Carelon theme
<ThemeProvider theme="carelon">
  <App />
</ThemeProvider>

// Elevance theme
<ThemeProvider theme="elevance">
  <App />
</ThemeProvider>
```

### Custom Theme (Advanced)

You can also provide a custom theme object:

```tsx
import { ThemeProvider } from '@lean-ids/components';
import { carelonTheme } from '@lean-ids/tokens';

const customTheme = {
  ...carelonTheme,
  colors: {
    ...carelonTheme.colors,
    primary: {
      ...carelonTheme.colors.primary,
      500: '#your-custom-color',
    },
  },
};

<ThemeProvider theme={customTheme}>
  <App />
</ThemeProvider>
```

## 📱 TypeScript Support

Lean IDS is built with TypeScript and provides complete type definitions.

### Importing Types

```tsx
import { 
  ButtonProps, 
  InputFieldProps, 
  CheckboxProps 
} from '@lean-ids/components';

// Use types in your components
const MyButton: React.FC<ButtonProps> = (props) => {
  return <Button {...props} />;
};
```

### Type-Safe Props

All components have fully typed props with IntelliSense support:

```tsx
import { Button } from '@lean-ids/components';

<Button
  variant="primary" // TypeScript will suggest: 'primary' | 'outlined' | 'link' | 'success' | 'warning' | 'alert'
  size="medium"     // TypeScript will suggest: 'xs' | 'small' | 'medium' | 'large' | 'xl'
  disabled={false}  // TypeScript knows this is boolean
  onClick={() => {}} // TypeScript knows the signature
>
  Click Me
</Button>
```

## ♿ Accessibility Best Practices

### Always Provide Labels

```tsx
// ✅ Good - has label
<InputField label="Email Address" type="email" />

// ❌ Bad - no label
<InputField type="email" />
```

### Use Semantic HTML

```tsx
// ✅ Good - button for actions
<Button onClick={handleSubmit}>Submit</Button>

// ❌ Bad - div with onClick
<div onClick={handleSubmit}>Submit</div>
```

### Provide Alternative Text

```tsx
// ✅ Good - aria-label for icon-only buttons
<Button 
  leadingIcon={<CloseIcon />}
  showLabel={false}
  aria-label="Close dialog"
/>
```

### Handle Keyboard Navigation

All Lean IDS components support keyboard navigation by default:
- **Tab** - Move focus
- **Enter/Space** - Activate buttons and checkboxes
- **Escape** - Close modals and dropdowns (when available)

## 🔧 Common Patterns

### Form with Validation

```tsx
import { InputField, Button } from '@lean-ids/components';
import { useState } from 'react';

function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [errors, setErrors] = useState({});

  const validate = () => {
    const newErrors = {};
    if (!email) newErrors.email = 'Email is required';
    if (!password) newErrors.password = 'Password is required';
    setErrors(newErrors);
    return Object.keys(newErrors).length === 0;
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    if (validate()) {
      // Submit form
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <InputField
        label="Email"
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        state={errors.email ? 'error' : 'default'}
        helperText={errors.email}
      />
      
      <InputField
        label="Password"
        type="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        state={errors.password ? 'error' : 'default'}
        helperText={errors.password}
      />
      
      <Button type="submit" variant="primary">
        Sign In
      </Button>
    </form>
  );
}
```

### Toast Notifications

```tsx
import { Toast } from '@lean-ids/components';
import { useState } from 'react';

function NotificationExample() {
  const [toast, setToast] = useState(null);

  const showSuccess = () => {
    setToast({
      type: 'success',
      message: 'Operation completed successfully'
    });
    setTimeout(() => setToast(null), 3000);
  };

  return (
    <>
      <Button onClick={showSuccess}>Show Success</Button>
      
      {toast && (
        <Toast
          type={toast.type}
          text={toast.message}
          onClose={() => setToast(null)}
        />
      )}
    </>
  );
}
```

## 🐛 Troubleshooting

### Styled Components Version Conflict

If you see warnings about multiple styled-components versions:

```bash
npm dedupe styled-components
```

### Theme Not Applied

Make sure `ThemeProvider` wraps all components:

```tsx
// ✅ Correct
<ThemeProvider theme="carelon">
  <Button>Click Me</Button>
</ThemeProvider>

// ❌ Wrong - Button outside ThemeProvider
<Button>Click Me</Button>
<ThemeProvider theme="carelon">
  {/* other content */}
</ThemeProvider>
```

### TypeScript Errors

Ensure you have the correct TypeScript version:

```bash
npm install --save-dev typescript@^5.7.0
```
